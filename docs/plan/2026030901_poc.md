# PoC Plan — 2026-03-09-01

> Prove the core chain: programmatic trigger -> OpenCode (headless) -> MCP policy proxy -> hosted MCP server -> result.

## Decision Log

| #   | Decision                                      | Rationale                                                                                            |
| --- | --------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| D1  | **OpenCode** as agent runtime                 | Best coding capability, headless server + SDK, MIT licensed. Orchestration layer built externally.   |
| D2  | **Atlassian MCP** as first integration target | Hosted endpoint (`mcp.atlassian.com/v1/mcp`), no infra to run, read-heavy (low risk).                |
| D3  | **Skip real triggers** for PoC                | A `curl` POST replaces Slack/cron/webhook triggers. Proves the chain without OAuth or webhook infra. |
| D4  | **Skip approval flow** for PoC                | Proxy enforces allow/block only. Async approval queue deferred to MVP.                               |
| D5  | **Skip Slack, PostHog, GitHub** for PoC       | One integration is enough to validate proxy passthrough. Others are additive.                        |
| D6  | **Local Docker Compose** for deployment       | Fastest iteration. No Railway setup. Easy to tear down.                                              |
| D7  | **Monorepo with pnpm workspaces**             | Two packages (`runner`, `proxy`) share config. Simple, no Turborepo needed yet.                      |
| D8  | **Minimal custom proxy** (no framework)       | Express + raw HTTP fetch to downstream MCP. Keeps dependencies minimal.                              |

## Phases

### Phase 1 — Project Init

**Goal**: Repo skeleton that builds and runs `hello world` in both packages.

Steps:

1. Root `package.json` with pnpm workspaces
2. Shared `tsconfig.base.json`
3. `packages/proxy/` — Express server, health endpoint, TypeScript build
4. `packages/runner/` — Express server, health endpoint, TypeScript build
5. Root scripts: `pnpm build`, `pnpm dev`
6. `.gitignore`
7. Verify: `pnpm install && pnpm build` succeeds, both servers start and respond to `/health`

**Exit criteria**: `curl localhost:3001/health` (proxy) and `curl localhost:3000/health` (runner) both return 200.

---

### Phase 2 — MCP Policy Proxy

**Goal**: Proxy receives MCP Streamable HTTP requests, evaluates policy, forwards allowed requests to Atlassian MCP, returns responses.

Steps:

1. MCP Streamable HTTP handler (POST `/mcp` endpoint)
2. Policy config file (`proxy.policy.json`) — allowlist of tool names, default deny
3. Request forwarding to downstream MCP server via HTTP
4. Response passthrough back to caller
5. Structured JSON logging of every request (tool name, policy decision, timestamp)
6. Error handling (downstream timeout, auth failure, malformed request)

**Exit criteria**: Send a raw MCP `tools/list` request to the proxy, see it forwarded to Atlassian MCP and tools returned. Send a `tools/call` for an allowed tool, see it succeed. Send a `tools/call` for a blocked tool, see it rejected with a policy message.

**Test**: Script that sends MCP JSON-RPC requests via `curl` to the proxy.

---

### Phase 3 — Runner (OpenCode Headless + Trigger)

**Goal**: HTTP trigger spawns an OpenCode headless session pointed at the proxy, sends a prompt, returns the result.

Steps:

1. `POST /trigger` endpoint accepting `{ "prompt": "..." }`
2. OpenCode headless process management (start/reuse `opencode serve`)
3. Configure OpenCode to use the proxy as its MCP server
4. Send prompt via `@opencode-ai/sdk`, stream response
5. Return agent response as HTTP response body
6. Timeout and error handling

**Exit criteria**: `curl -X POST localhost:3000/trigger -d '{"prompt":"List available tools"}'` returns the tool list from Atlassian MCP (via proxy).

**Test**: Manual curl + verify proxy logs show the forwarded requests.

---

### Phase 4 — Docker Compose + End-to-End

**Goal**: All components run in containers, full chain works with one command.

Steps:

1. `Dockerfile` for proxy
2. `Dockerfile` for runner (includes OpenCode binary)
3. `docker-compose.yml` wiring both services + network
4. Environment variable handling (Atlassian auth injected at runtime)
5. End-to-end test script

**Exit criteria**: `docker compose up` starts both services. From host: `curl -X POST localhost:3000/trigger -d '{"prompt":"List open issues in ACME project"}'` returns real Jira data through the full chain.

**Test**: `scripts/test-e2e.sh` that runs the curl and validates output.

#### Docker Setup

```bash
# 1. Start services (no auth setup needed — uses opencode/big-pickle, free model)
ATLASSIAN_BASIC_AUTH=<base64-encoded-email:token> docker compose up --build -d

# 2. Run e2e test from host
./scripts/test-e2e.sh
```

**Note**: OpenCode config is baked into the runner image at the global level (`~/.config/opencode/opencode.json`), using `opencode/big-pickle` (free, no auth required). A named volume (`opencode-data`) persists sessions and logs across container restarts.

- Worklog / memory persistence
