# Plan: Replace MCP with HTTP exec proxy for git/gh

**Date**: 2026-03-16
**Status**: Complete (all phases)

## Goal

Remove the GitHub MCP and git MCP integrations from OpenCode. Replace them with local-looking `git` and `gh` wrapper commands in the OpenCode container that forward over HTTP to the authenticated `remote-cli` container.

## Why

- The MCP transport layer adds complexity (session management, SDK dependency, proxy config) for what is fundamentally "run a command and return output".
- Adding `gh` CLI support via MCP would mean another MCP server or tool definition + proxy config. A simple HTTP exec endpoint is far lighter.
- Wrapper scripts let the agent use `git` and `gh` as if they were local binaries ‚Äî no MCP tool syntax needed.

## Phases

### Phase 1 ‚Äî HTTP exec service + policy

Convert the existing `git-mcp` package into a plain HTTP exec service renamed `remote-cli`, with server-side policy.

1. Rename `packages/git-mcp` ‚Üí `packages/remote-cli` (package name, tsconfig, Dockerfile target, docker-compose service, entrypoint)
2. Replace MCP transport with `POST /exec/git` and `POST /exec/gh`
   - JSON request: `{ args, cwd }` ‚Üí JSON response: `{ stdout, stderr, exitCode }`
   - Keep `/health`
   - Remove `@modelcontextprotocol/sdk` dependency
3. Add server-side command policy (`packages/remote-cli/src/policy.ts`)
   - Validate `cwd` is absolute and under `/workspace/repos` or `/workspace/worktrees`
   - Block `git clone` and `git init`
   - Allow-list for `gh` subcommands (pr, issue, repo, run, workflow, release)
   - Block `gh api` entirely (specific commands cover all needed operations)
4. Unit tests in `packages/remote-cli/src/policy.test.ts`

**Exit criteria:**

- [ ] `corepack pnpm install` succeeds
- [ ] `corepack pnpm -r build` succeeds
- [ ] `corepack pnpm -r typecheck` succeeds
- [ ] `corepack pnpm test` passes (including new policy tests)
- [ ] `remote-cli` serves `/exec/git` and `/exec/gh` with policy enforcement

### Phase 2 ‚Äî Docker + wrappers

Install `gh` CLI in the container and add OpenCode wrapper scripts.

1. Install `gh` in the `remote-cli` Dockerfile target
2. Update `entrypoint.sh` to export `GH_TOKEN` from the existing `GITHUB_PAT`
3. Add OpenCode wrapper scripts in `docker/opencode/bin/`:
   - `remote-cli.mjs` ‚Äî shared HTTP client
   - `git` ‚Äî resolves `-C`, posts to `/exec/git`
   - `gh` ‚Äî posts to `/exec/gh`
4. Update `docker/opencode/Dockerfile` to copy bin/, add to PATH, set `THOR_REMOTE_CLI_URL`

**Exit criteria:**

- [ ] `gh --version` works inside remote-cli container
- [ ] Wrapper scripts exist and are executable
- [ ] OpenCode Dockerfile builds successfully

### Phase 3 ‚Äî Remove MCP integrations + update configs

Remove the old MCP plumbing and rewire docker-compose.

1. Remove `github` and `git` entries from `opencode.json`
2. Delete `proxy.github.json` and `proxy.git.json`
3. Update `PROXY_INSTANCES` to only 3010‚Äì3012 (atlassian, posthog, slack)
4. Remove ports 3013‚Äì3014 from proxy service; update healthcheck
5. Make `opencode` depend directly on `remote-cli`
6. Remove `GITHUB_PAT` from proxy environment (no longer needed there)

**Exit criteria:**

- [ ] `opencode.json` has no `github` or `git` MCP entries
- [ ] `proxy.github.json` and `proxy.git.json` are deleted
- [ ] docker-compose proxy only exposes ports 3010‚Äì3012
- [ ] `opencode` depends on `remote-cli` directly

### Phase 4 ‚Äî Update docs and agent instructions

1. Rewrite `docker/opencode/agents/build.md` ‚Äî agent uses `git`/`gh` commands instead of MCP tools
2. Update `docs/feat/mvp.md` to reflect new architecture
3. Update `README.md`

**Exit criteria:**

- [ ] `build.md` references `git`/`gh` CLI, not MCP tools
- [ ] `mvp.md` describes the HTTP exec architecture

## Out of scope

- Approval flow for gh commands (may add later)
- E2E testing against live Docker containers

## Decision Log

| #   | Decision                                                       | Rationale                                                                                                                 |
| --- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| 1   | Rename `git-mcp` ‚Üí `remote-cli`                                | The service is no longer an MCP server; name should reflect what it does.                                                 |
| 2   | Single `remote-cli.mjs` shared by both wrappers                | DRY ‚Äî both `git` and `gh` wrappers do the same HTTP POST, just to different endpoints.                                    |
| 3   | Shell wrappers (`git`, `gh`) call `node remote-cli.mjs`        | Lets them look like real binaries in PATH. Node handles JSON and HTTP natively.                                           |
| 4   | Policy is server-side only                                     | The wrappers are untrusted (inside agent container). All validation happens in remote-cli.                                |
| 5   | Block `gh api` entirely                                        | It's a backdoor around the command policy. All needed operations are covered by specific `gh` commands (pr, issue, etc.). |
| 6   | Add vitest to remote-cli for policy tests                      | Consistent with proxy package which already uses vitest.                                                                  |
| 7   | Export only `GH_TOKEN`, not both `GH_TOKEN` and `GITHUB_TOKEN` | `gh` CLI checks `GH_TOKEN` first. One env var is sufficient and clearer.                                                  |
