# ScoutQA CLI Integration

**Date**: 2026-03-17
**Status**: In Progress

## Goal

Give Kally the ability to run exploratory QA tests, accessibility audits, and smoke tests via ScoutQA. The original plan was to use ScoutQA's MCP server, but the endpoint requires OAuth that doesn't work with API keys. Instead, we expose the `@scoutqa/cli` as a remote binary (same pattern as `git` and `gh`).

Install `@scoutqa/cli` in the `remote-cli` container, add an `/exec/scoutqa` endpoint with argument validation, and add a wrapper script in the OpenCode container.

- Argument policy allows: `create-execution`, `send-message`, `list-executions`, `complete-execution`
- `SCOUT_API_KEY` flows from `.env` to `remote-cli` container

## Decision Log

| #   | Decision                             | Reason                                                            |
| --- | ------------------------------------ | ----------------------------------------------------------------- |
| 1   | CLI via remote binary, not MCP proxy | ScoutQA's MCP endpoint requires OAuth; API key only works via CLI |
| 2   | Reuse `remote-cli` service           | Same remote exec pattern as `git`/`gh`; no new container needed   |
| 3   | `SCOUT_API_KEY` env var              | CLI auto-detects this env var; no config file management needed   |

## Out of Scope

- Automated test scheduling via cron (can be added later)
- Custom MCP wrapper around the CLI
