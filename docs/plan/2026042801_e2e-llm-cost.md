# E2E LLM Cost Reduction

**Date**: 2026-04-28
**Status**: Implemented; revised after review; PR CI trigger updated
**Source**: `/workspace/runs/20260428-e2e-llm-cost/findings_1.md`

## Goal

Reduce the default Thor e2e path's real OpenCode/LLM inference cost while preserving confidence in the runner, gateway, and remote-cli boundaries.

The expensive behavior today is concentrated in `scripts/test-e2e.sh`, which mixes deterministic direct API checks with roughly a dozen `POST /trigger` prompts. The default `pnpm test:e2e` path should become mostly deterministic and cheap; real LLM/OpenCode coverage should remain available through an explicit smoke/manual path.

## Scope

- Split expensive OpenCode/LLM-backed e2e sections from deterministic direct checks.
- Keep a minimal, explicit LLM smoke path for end-to-end OpenCode reachability and wrapper re-entry.
- Replace removed default assertions with targeted deterministic runner coverage using a mocked OpenCode SDK/event bus where practical.
- Keep `scripts/test-sandbox-e2e.sh` as direct non-LLM integration coverage; runtime trimming of that script is separate work.

## Phases

### Phase 1 — Introduce deterministic runner `/trigger` coverage

Add focused tests around the runner trigger orchestration before moving behavior out of the default e2e path.

Implementation targets:

- Make the runner Express app testable without listening on a real port, preferably by extracting app construction/startup from `packages/runner/src/index.ts`.
- Mock `@opencode-ai/sdk` and OpenCode session APIs so tests do not perform inference.
- Cover at least:
  - correlation-key session creation and same-key resume;
  - stale stored session fallback, including prior-notes hint when notes exist;
  - busy resumed session with `interrupt` absent/false returning `{ busy: true }` before `promptAsync`;
  - busy resumed session with `interrupt: true` invoking abort and then sending the new prompt;
  - root/repo memory and tool-instruction prompt injection on new sessions, but not on normal resumed sessions;
  - notes creation/append summary enough to prove the replaced session-resume/memory e2e behavior still has deterministic coverage.

Exit criteria:

- Runner tests run under `pnpm test` with no OpenCode service and no LLM provider.
- Tests assert emitted NDJSON `start`/`done` fields (`sessionId`, `resumed`, `busy` where relevant) rather than relying on model text.
- No production behavior change except app/testability refactoring.

### Phase 2 — Split `scripts/test-e2e.sh` into default direct checks and explicit LLM smoke

Refactor the shell e2e script so `pnpm test:e2e` no longer runs the high-cost prompt matrix by default.

Implementation targets:

- Add a dedicated root script (`test:opencode-e2e`) for LLM-backed smoke coverage.
- Default `scripts/test-e2e.sh` should keep deterministic health, remote-cli `/internal/exec`, approval API, and git/gh policy checks.
- Replace the current GH auth and worktree prompts with direct remote-cli/internal exec checks already present or with a direct `git worktree add` check through the appropriate trusted path.
- Move high-cost/nondeterministic sections out of the default path: cross-session phrase memory, full busy/interrupt stress, alias phrase recall through cron, and extra GH/worktree model prompts.

Exit criteria:

- `pnpm test:e2e` can run without a real LLM provider and does not call `/trigger`.
- Deterministic direct sections still fail loudly when remote-cli/gateway prerequisites are missing.
- Script output clearly reports which LLM smoke sections are skipped and how to enable them.

### Phase 3 — Add minimal explicit LLM smoke path

Keep one small end-to-end path for OpenCode/runtime integration, but make its cost intentional.

Implementation targets:

- The explicit LLM smoke should cover only the highest-value full-chain signals:
  - one cheap wrapper/tool command via the agent;
  - one same-correlation resume prompt; and
  - one approval-status re-entry prompt if an approval-required tool is discoverable.
- Prefer reusing helper functions from `scripts/test-e2e.sh` rather than duplicating large shell blocks.
- Ensure the root `package.json` exposes an obvious command for this path if the flag alone is not sufficient.

Exit criteria:

- Operators can intentionally run the smoke with a documented command.
- The smoke uses materially fewer model turns than the current script and excludes nightly/manual stress scenarios.
- Default CI/PR e2e remains deterministic unless a workflow explicitly opts in.

### Phase 4 — Verification, docs, and PR readiness

Implementation targets:

- Update README/package script documentation if commands or flags change.
- Run targeted unit tests for the new runner coverage, then root `pnpm test` and `pnpm typecheck` as feasible.
- Run deterministic e2e locally if the required services are available; otherwise document skipped service-dependent verification.
- Push the branch to trigger the appropriate GitHub workflow before opening the PR, per `AGENTS.md`.

Exit criteria:

- Unit/typecheck evidence is recorded in the PR body.
- Deterministic e2e path is shown not to require LLM inference.
- PR description calls out the remaining explicit LLM smoke command and any intentionally deferred nightly/manual coverage.

| D-1 | Create a plan file for this work                                                                                             | The change spans runner tests, shell e2e structure, scripts/docs, and PR workflow; phase boundaries help preserve coverage while reducing cost.                                          |
| D-2 | Add deterministic runner coverage before deleting default LLM assertions                                                     | The investigation identified runner `/trigger` as the biggest coverage gap; moving e2e prompts first would reduce confidence around resume, busy/interrupt, memory injection, and notes. |

| D-7 | Keep the OpenCode LLM smoke in Core E2E, gated away from `pull_request` and to `refs/heads/main` only                        | PR events should run cheap deterministic checks, while main still exercises the full runner/OpenCode/LLM path in the existing e2e stack.                                                 |

## Out of Scope

- Optimizing Daytona/sandbox e2e runtime.
- Rewriting remote-cli policy tests beyond direct replacement checks needed by this split.
- Adding nightly workflow scheduling unless a later implementation phase needs it.
- Changing OpenCode SDK/server versions.
